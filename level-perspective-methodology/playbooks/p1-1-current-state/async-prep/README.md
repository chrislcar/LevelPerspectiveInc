## Canonical structure (inherited across specs)

Each async-prep spec follows the canonical 5-field schema:

1. **What this step is** — one paragraph
2. **Objective** — what the async output enables
3. **How this step achieves its objective** — numbered list of mechanics
4. **Deliverable produced** — what the methodologist receives 24h pre-session
5. **Coaching role** — what the methodologist does pre-session and during

Plus inheritance from the canonical pattern:
- Strategyzer canonical DO-step structure (title rail, instructions, example, working canvas, completion mechanic)
- Async select-then-work (pick 3 of 6 + name your own)
- Binary green/yellow sticky discipline (max 8 words)
- Four-zone working card structure (What we found / Sources / Date range / What it doesn't tell you)
- Per-lane named owner with 3+ card domain coverage
- Quality bar footer
- PathLine reference example
- Methodologist surfaces / leadership owns / AI structures
