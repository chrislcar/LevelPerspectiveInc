## Why a fictional reference org

Cross-step continuity. The same fictional company appears in:
- Strategic Direction async prep example (Aspiration / Arenas / Financial Performance Philosophy)
- Brand Image async prep example (External Perception / What We Stand For)
- Organizational Culture async prep example (Behaviors / Enablers-Blockers)
- Center reconstruction live demo
- Cross-element gap synthesis example
- Learning video worked examples

A consistent reference org lets stakeholders pattern-match across steps without re-grounding in a new fictional context every time.

## Data discipline

PathLine evidence in examples is realistic but fictional. Names of real companies, real comp plans, or real customer interviews never appear here. If a worked example needs more specificity than fiction supports, that's a sign the example should reference an anonymized real engagement (and that lives in SharePoint engagement folders, not here).
