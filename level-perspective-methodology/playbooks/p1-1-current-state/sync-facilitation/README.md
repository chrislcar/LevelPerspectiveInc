## Canonical structure (inherited across specs)

Each sync facilitation spec follows the canonical 5-field schema:

1. **What this step is** — one paragraph
2. **Objective** — what gets committed; what stays unresolved
3. **How this step achieves its objective** — minute-by-minute facilitation script with named technique per move
4. **Deliverable produced** — what content lands in Slide 1 (or appendix) for that element
5. **Coaching role** — methodologist's neutrality discipline per move

Move-level structure:
- Time budget per move
- Named facilitation technique (ORID / 1-2-4-All / Min Specs / Wicked Questions / Consensus Workshop)
- Sub-step number (mapped to canonical SGF Briefing path where applicable)
- Output per move
- Methodologist neutrality test per move

## Variant relationship

Virtual variants are authored first; in-person variants inherit and adapt for time and energy budget. In-person sessions combine multiple steps that would be separate virtual sessions.
